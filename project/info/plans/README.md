# Plans

This folder is for implementation plans, task breakdowns, and hand-over notes.

Use it for:

- plans for non-trivial features
- active task checklists
- hand-over notes for unfinished work
- short status summaries for ongoing threads

Keep plans tidy:

- prefer one Markdown file per task or feature
- update status when the plan changes
- mark stale or abandoned plans clearly
- remove duplicated notes when the source of truth lives elsewhere

Start from `HANDOVER_TEMPLATE.md` when unfinished work needs to be resumed by someone else later.

The goal is to make ongoing work easy to resume later, by either a human or an agent.
