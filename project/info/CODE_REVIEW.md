# Code Review Checklist

Prioritize real bugs, regressions, missing tests, and unclear behavior.

## Check for:

- behavior that does not match the request
- missing or weak tests
- accidental scope creep
- brittle logic or unclear edge cases
- naming or structure that makes the change harder to understand
- docs that should change but did not

## A good review comment should:

- point to a real risk
- explain why it matters
- suggest a concrete improvement when possible
